# ログの重要ポイントを見逃さない！ブックマーク機能ガイド 🔖

大量のログを調査しているとき、「あ、ここ怪しいな」「ここがエラーの起点だ」と思った箇所を忘れてしまったことはありませんか？
MultiLogViewer の **ブックマーク機能** を使えば、ログに「しおり」を挟んで、後から自由自在に振り返ることができます。

---

## 1. 基本の「キ」：しおりを挟む・外す 📍

操作はとっても簡単です。

### 💡 やりかた

気になるログの行を選んで、キーボードの **`Ctrl + B`** を押すだけ！

- **付ける**: 左端に「しおりアイコン」が表示されます。
- **外す**: もう一度 `Ctrl + B` を押すと消えます。

これだけで、数万行あるログの中から「大事な行」をいつでも見つけられるようになります。

---

## 2. 挟んだしおりをスイスイ辿る 🏃‍♂️

しおりをいくつか挟んだら、それらを順番に確認してみましょう。

### 💡 ナビゲーションキー

- **`F2` キー**: 次のブックマークへジャンプ。
- **`Shift + F2` キー**: 前のブックマークへ戻る。

マウスでスクロールして探す必要はありません。キーを叩くだけで、調査対象の行だけをピョンピョンと飛び回れます。

---

## 3. ブックマークパネルで「メモ」を残す 📝

画面右上のブックマークアイコンをクリックすると、**ブックマーク専用のパネル**が開きます。

### ✨ ここでできること

1. **一覧表示**: 今ブックマークしている行がリストで並びます。クリックすればその行に即ジャンプ！
2. **メモ（MEMO）入力**: 選択したブックマークに対して、「ここで通信断」「DB 再起動」のようなメモを残せます。
   - あとで「このブックマーク、なんで付けたんだっけ？」となるのを防げます。
3. **一括解除**: 左端のアイコン列を右クリックして「ブックマークをすべて解除」を選べば、一気にリセットできます。

---

## 4. 【上級編】タイムラインで「犯人」を追い詰める！ 🕵️‍♀️

ブックマークの真の力は、パネルの右上にある「三本線（ダイジェスト）」ボタンを押したときに発揮されます。

### 🚀 ダイジェスト・ビュー

ブックマークした行だけを並べた、特別な分析画面が開きます。

| 項目           | 説明                                                               |
| :------------- | :----------------------------------------------------------------- |
| **Delta (Δt)** | **前のブックマークから何秒（何分）経ったか** を表示します。        |
| **Cumulative** | 最初のブックマークから合計でどれくらい時間が経ったかを表示します。 |

### 💡 具体的な使いかた例

「サーバーが重くなった」というトラブルを調査するとします。

1. 「CPU が上がり始めた行」にブックマーク 📌
2. 「最初のタイムアウトエラー」にブックマーク 📌
3. 「完全に停止した行」にブックマーク 📌
4. **ダイジェストを表示！**
   - 「CPU 上昇からエラーまで **+00:02:30**」
   - 「エラーから停止まで **+00:00:15**」
   - **「なるほど、異常開始から 2 分半で予兆が出て、そこから 15 秒で死んだんだな」**

といった、時間軸での鋭い分析がパッとできるようになります！

---

## 5. ブックマークだけで絞り込む 🔍

他のログが邪魔なときは、フィルターメニューから **「Bookmark フィルターを追加」** を選んでみてください。
ブックマークした行 **だけ** が画面に表示されるようになり、集中して分析できます。

---

💡 **ヒント**: ブックマークはアプリを閉じると消えてしまいます（メモリ上のみ）。大事な分析結果は、メモ機能とダイジェスト画面を活用して、忘れないうちに調査報告書にまとめましょう！
